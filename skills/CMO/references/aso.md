# App Store Optimization — CMO Reference

## When to Load
Load this reference when:
- The user is shipping an app to the App Store or Google Play
- Optimizing app store listing, screenshots, or metadata
- Planning an app launch or improving download conversion
- The user asks about ASO, app store listing, or app marketing

## Keyword Strategy Framework

### Keyword Tiers

```
Primary Keywords (High Volume, High Relevance):
- [Keyword]: Volume estimate, Competition level, Relevance: X/10

Long-tail Keywords (Lower Volume, Higher Intent):
- "[Specific phrase]": Targets specific use case, lower competition

Competitive Gaps:
- Keywords competitors rank for that we don't
- Underutilized keywords with growth potential
- Emerging terms with low competition
```

## Metadata Optimization

### App Title
- **iOS:** [Primary Keyword] - [Value Proposition] (30 char limit)
- **Android:** [Primary Keyword]: [Secondary Keyword] [Benefit] (50 char limit)

### Subtitle / Short Description
- **iOS Subtitle:** [Key Feature] + [Primary Benefit] + [Target Audience] (30 chars)
- **Android Short:** Hook + Primary Value Prop + CTA (80 chars)

### Long Description Structure
1. **Hook** — Problem/solution statement (first 3 lines visible before "more")
2. **Key Features & Benefits** — Bulleted, benefit-led (not feature-led)
3. **Social Proof** — Ratings, download count, awards, press mentions
4. **Use Cases** — Who it's for, specific scenarios
5. **Call to Action** — Download prompt
6. **Keyword Integration** — Natural placement, no stuffing

## Screenshot Sequence Strategy

Each screenshot has a specific job:

| Position | Purpose | Content |
|----------|---------|---------|
| 1 (Hero) | Immediate value proposition | Key feature demo + benefit headline + visual appeal |
| 2-3 | Core features | Primary use case walkthrough + user benefit copy |
| 4-5 | Supporting features | Feature depth + use case variety + competitive advantages |
| 6+ | Social proof / edge cases | Reviews, awards, additional features |

### Screenshot Best Practices
- Headline text above device frame (benefit, not feature name)
- Actual app UI visible (not mockups or illustrations)
- Consistent visual style across all screenshots
- First screenshot does the most work — treat it like a billboard

## App Preview Video

### Structure (15-30 seconds)
- **0-3s (Hook):** Problem statement or compelling question, pattern interrupt
- **3-20s (Demo):** Core functionality with real scenarios, smooth transitions
- **20-30s (CTA):** Clear next step, value reinforcement, brand moment

### Technical Specs
- iOS: 15-30 seconds, up to 3 previews per localization
- Autoplay without sound — must work silently
- Captions/text overlays required for sound-off viewing

## App Icon A/B Testing Variables
- Color schemes (primary brand vs. category-optimized)
- Complexity (minimal vs. detailed)
- Text inclusion (none vs. abbreviated)
- Symbol vs. literal representation

## A/B Testing Roadmap
1. **Phase 1:** Icon and first screenshot (highest impact)
2. **Phase 2:** Description and keyword optimization
3. **Phase 3:** Full screenshot sequence optimization

## Success Targets
- 30%+ month-over-month organic download growth
- Top 10 rankings for 20+ relevant terms
- 25%+ conversion rate improvement through optimization
- 4.5+ star rating maintained
- Featured by Apple/Google (work toward editorial features)

## Apple-Specific Tips
- Keywords field: 100 characters, comma-separated, no spaces after commas
- Promotional text (170 chars) can be updated without review — use for timely messaging
- In-App Events appear in search results — use for visibility
- App Clips can serve as try-before-download conversion tool
